ответы на контрольные вопросы:
1. ООП — парадигма программирования, где программа строится как набор взаимодействующих объектов, каждый из которых является экземпляром определенного класса. Решает задачи моделирования реального мира, повышения переиспользуемости, читаемости и поддерживаемости кода.

2. Основные принципы ООП:

Инкапсуляция — объединение данных и методов в единый объект, скрытие внутренней реализации.

Наследование — создание новых классов на основе существующих с возможностью расширения функциональности.

Полиморфизм — возможность использования объектов разных классов через единый интерфейс.

Абстракция — выделение существенных характеристик объекта и игнорирование несущественных.

3. Класс — шаблон (описание) для создания объектов, определяющий их структуру и поведение. Объект — конкретный экземпляр класса, созданный по его шаблону.

4. Наследование в Python реализуется через указание родительского класса в определении дочернего. Используется для создания иерархий классов, повторного использования кода и расширения функциональности.

5. Инкапсуляция — сокрытие внутреннего состояния объекта и деталей реализации. В Python атрибуты скрываются через префикс _ (protected) или __ (private) для предотвращения прямого доступа и контроля над изменением данных.

6. Полиморфизм — возможность объектов разных классов реагировать на одинаковые методы по-разному. Реализуется через переопределение методов в дочерних классах или использование интерфейсов.

7. Магические методы — специальные методы с двойными подчеркиваниями, которые вызываются неявно при определенных операциях. __str__ определяет строковое представление объекта при вызове str() или print().

8. Классовые атрибуты принадлежат самому классу, общие для всех экземпляров. Атрибуты экземпляра принадлежат конкретному объекту и уникальны для каждого экземпляра.

9. super() обеспечивает корректный доступ к методам родительского класса при множественном наследовании, помогает избежать дублирования кода и проблем с инициализацией.

10. Взаимодействие объектов обеспечивается через передачу объектов как параметров, вызов методов других объектов, композицию и агрегацию. Например, объект одного класса может использовать методы объекта другого класса для выполнения задачи.
